# NexCubic — landing & multi-page website design (ready-to-build)

Awesome — below is a complete, ready-to-use plan for a modern, mobile-first, Tailwind-based multipage website for **NexCubic** (tech support for startups & companies: web & app development, AI automation, UI/UX, marketing & advertising, tech consultancy). I include: sitemap, detailed page blueprints, UI/UX & animation notes, Tailwind-ready HTML component snippets (hero, services grid, blog list, internship listing, contact), accessibility/SEO checklist, and **a ready-to-run “Bolt form” command** (two variants: a Bolt-CLI style and a `curl` POST with JSON payload) you can paste into your automation to generate the site.

---

1. **Homepage** — hero, value props, services overview, trust logos, case studies, CTA

---

# Visual style & inspiration

* **Base**: Tailwind CSS utility-first, modern layout system (grid + container).
* **Influence**: 21st.dev (clean, readable, generous spacing), Tailwind UI patterns, and “Loveable” — warm micro-interactions and friendly copy.
* **Palette (example)**:

  * Primary: `#0EA5A4` (teal/cyan)
  * Accent: `#7C3AED` (deep violet)
  * Dark: `#0F172A`
  * Light surfaces: `#F8FAFC`
* **Typography**: Inter / Poppins for headings, system font for body.
* **Roundedness**: `rounded-2xl` cards, soft shadows.
* **Animations**: subtle entrance (fade+translate), hover lift (translateY -4px + shadow), animated gradient on hero CTA. Use Framer Motion for React or `@tailwindcss/animate` CSS for plain HTML.

---

# Key UX principles

* Mobile-first: single-column mobile wireframe, large tappable CTAs, sticky bottom CTA on mobile (Hire / Get Quote).
* Fast: minimal blocking scripts, preconnect to CDN, lazyload images.
* Clear hierarchy: immediate statement of who you serve and primary CTA within fold.
* Trust: logos, short case studies, testimonials, measurable outcomes.
* Accessibility: semantic HTML, ARIA labels for dynamic content, 4.5:1 contrast for CTAs.

# Animations & interactions (technical notes)

* **Entrance**: Use `opacity` + `translateY(8px)` on elements with staggered delays (0.06s increments). In React use Framer Motion `AnimatePresence` and `motion.div`.
* **Hero CTA**: animated gradient using CSS `background-size` animation + subtle scale on hover.
* **Microinteractions**: button lift on hover (`transform: translateY(-4px)`), service card hover reveal (icon -> detail).
* **Loading**: skeletons for blog list and internships when fetching.
* **Performance**: use `prefers-reduced-motion` to disable animations for users who opt-out.

---

# SEO & accessibility checklist

* Unique `<title>` and `<meta description>` per page.
* Structured data for organization + blog posts (JSON-LD).
* Semantic elements `<main>`, `<header>`, `<nav>`, `<article>`.
* All images with `alt`.
* Forms have `aria-*` where needed; labels visible or `sr-only`.
* Mobile viewport meta and fast LCP (optimize hero image).

# Deployment + tech stack (recommended)

* **Static site / hybrid**: Next.js (app dir) + Tailwind + Framer Motion + MDX for blog.
* **Hosting**: Vercel or Netlify.
* **CMS**: Contentful / Sanity / Ghost if you want a UI for blog, or use MDX + GitHub for content.
* **Forms**: serverless functions (`/api/contact`) + Postmark for email, or use Formspree.
* **Analytics**: Plausible or Google Analytics (consent banner).
* **CI**: GitHub Actions; auto-deploy on main.

# “Bolt form” command to generate website

Below are two variants you can adapt. These are templates — edit fields (brand color, pages, logo path). If you already have a Bolt tool that accepts form JSON, paste the JSON into your Bolt generator or use the CLI example.

---

### A) Bolt-CLI style (example)

```bash
bolt create-site \
  --name "NexCubic" \
  --template "tailwind-nextjs" \
  --pages "home,services,blog,careers,about,contact" \
  --primary-color "#0EA5A4" \
  --accent-color "#7C3AED" \
  --logo "/assets/logo.svg" \
  --font "Inter:400,600,800" \
  --features "framer-motion,mdx,seo-schema,contact-form" \
  --output "./nexcubic-site"
```

> Paste into your terminal where `bolt` CLI exists — it will scaffold a Tailwind + Next.js starter with the pages listed and included Framer Motion and MDX.

---

### B) `curl` POST — JSON "Bolt form" (for an HTTP API)

```bash
curl -X POST "https://api.bolt.example.com/v1/sites/generate" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "projectName": "NexCubic",
    "template": "tailwind-nextjs",
    "pages": [
      {"path": "/", "type":"home"},
      {"path":"/services", "type":"list"},
      {"path":"/blog", "type":"blogIndex"},
      {"path":"/careers", "type":"careers"},
      {"path":"/about", "type":"about"},
      {"path":"/contact", "type":"contact"}
    ],
    "brand": {
      "logo": "/assets/logo.svg",
      "primaryColor": "#0EA5A4",
      "accentColor": "#7C3AED",
      "font": "Inter"
    },
    "features": ["framer-motion","mdx-blog","seo-jsonld","contact-form","responsive"],
    "deploy": {"provider":"vercel", "autoDeploy": true}
  }'
```

* Replace `https://api.bolt.example.com` with your bolt provider endpoint.
* Replace `YOUR_API_KEY` with your API key.
* The API should return a zip or Git repo link you can clone/deploy.
